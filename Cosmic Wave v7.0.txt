CosmicWave v7 — Hardened Kernel Codebase

A bare-metal x86-64 kernel in no_std Rust.

Improvements in v7

Kernel threads get private page tables — spawn_kthread now calls new_kthread_pml4() which allocates a fresh PML4 and copies only the upper-half kernel mappings (indices 256–511). Each kernel thread therefore runs in its own address space; a wild pointer in one thread cannot corrupt another thread's or a user process's mappings.

Guard pages + stack canaries — every kernel stack is allocated with one unmapped guard page below it (STACK_GUARD_PAGES = 1). A random 64-bit canary value (STACK_CANARY) is written at the top-of-stack sentinel slot at spawn time and verified on every schedule() switch-out. A corrupted canary triggers an immediate kernel panic before control returns anywhere.

Kernel .text read-only after boot — protect_kernel_text() is called just before irq_enable() in _start. It walks the boot page tables and clears PTE_W on every PTE covering the virtual range [__text_start, __text_end). The linker script exports these symbols. Subsequent attempts to write to kernel code fault with #PF.

Wait-queue / futex-like primitive — PipeInner's ad-hoc Weak<TaskInner> waker slots are replaced by a proper WaitQueue struct. WaitQueue holds a Spinlock<VecDeque<Weak<TaskInner>>>. Callers use wq.wait(guard) to atomically release a held spinlock, block the current task, and re-acquire on wakeup; wq.wake_one() and wq.wake_all() pop and re-enqueue waiters. Pipes use two wait queues (readers, writers). The same primitive is reused for sys_waitpid.

Syscall argument validation — a SyscallArgs helper struct is built at the top of every syscall from the interrupt frame. Every argument goes through typed validation functions (va_user, va_user_buf, fd_valid, sig_valid, len_nonzero, prot_valid, flags_valid) before use. Any invalid argument returns EINVAL / EBADF immediately without touching kernel state.
