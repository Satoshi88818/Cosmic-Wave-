CosmicWave v3 — Full Kernel Codebase

A bare-metal x86-64 kernel in no_std Rust featuring:

Higher-half kernel mapped at 0xFFFF_8000_0000_0000

Real 4-level page tables with per-process user page tables

ACPI/MP table parser — enumerates all logical CPUs and their APIC IDs from RSDP → RSDT/XSDT → MADT, with MP table fallback

Demand paging — anonymous pages faulted in on first access; CoW fork; per-VMA red-black tree; SIGSEGV on unmapped access

Exception table (__ex_table) — safe user-memory accessors with per-instruction fixup entries; no more kill-on-fault

Copy-from-user / copy-to-user with SMAP-aware safe accessors backed by the exception table

Per-CPU TSS with dedicated IRQ stacks and IST stacks (NMI / #DF / #MC / #PF)

SMP bringup (INIT–SIPI–SIPI) for however many APs ACPI reports (up to MAX_CPUS)

Cooperative round-robin scheduler with callee-save context switching

POSIX signals (sigaction / sigreturn trampoline)

Hardened ELF loader — all previous bugs fixed, p_align enforced

Target: QEMU -smp 2 (or more) via build.sh.

Design Notes

ACPI/MP Parser

The parser follows the standard ACPI boot flow:

RSDP scan — searches 0xE0000–0xFFFFF (BIOS ROM) and the EBDA for the "RSD PTR " signature, validating the 20-byte checksum.

RSDT vs XSDT — if the RSDP revision is ≥ 2 and an XSDT address is present, entries are 8-byte physical pointers; otherwise RSDT uses 4-byte pointers.

MADT walk — iterates variable-length interrupt controller structures, extracting type-0 (Processor Local APIC) entries with the Enabled flag set. Each APIC ID is stored in APIC_IDS[].

MP fallback — if ACPI is absent, the Intel MP Floating Pointer Structure (_MP_) is scanned in the BIOS ROM and EBDA. Processor entries (type 0) are parsed for usable APIC IDs.

SMP bringup — smp_bringup() now iterates over the APIC_IDS array rather than hardcoding APIC ID 1, supporting any number of APs up to MAX_CPUS.

Exception Table

The .ex_table section (defined in the linker script) holds (fault_va, fixup_va) pairs as consecutive u64 values. The #PF handler calls extable_fixup(fault_rip) before any demand-paging or kill logic:

#PF fires at IP X → extable_fixup(X) searches .ex_table → found: set frame.RIP = fixup_va, return from handler → not found: proceed to CoW / demand paging / SIGSEGV 

Each safe accessor (copy_from_user, copy_to_user, get_user_u64, put_user_u64) is a #[naked] function whose rep movsb / mov instruction is the registered fault site. The fixup path re-arms SMAP and returns a non-zero error code, which the Rust wrapper converts to Err(()).

Demand Paging

Every process has a BTreeMap<usize, VmaRegion> (the vmas field in TaskInner). A VMA records:

base / end — page-aligned virtual address range

prot — PROT_READ | PROT_WRITE | PROT_EXEC combination

flags — PTE flags to install (PTE_P | PTE_U | ...)

Pages within a VMA are not allocated at mmap time — they are allocated by the #PF handler on first access. The sys_mmap syscall (number 9) only inserts a VMA entry. sys_munmap (number 11) removes VMA entries and frees any committed pages.

ELF segments are eagerly mapped (committed) at load time; the stack VMA covers 2 MiB below USER_STACK_TOP and grows down on demand.

Copy-on-Write Fork

sys_fork performs a full page-table walk of the parent's user half:

Allocates fresh PDPT / PD / PT frames for the child (table structure only).

For each present leaf PTE: if PTE_W is set, clears PTE_W and sets PTE_COW in both parent and child PTEs.

Issues invlpg for each modified VA in the parent's address space (single-CPU TLB flush; the broadcast IPI path is wired but not called in the fork hot-path to keep latency low).

On a write fault, the #PF handler detects err & 3 == 3 (present + write) with PTE_COW set, allocates a new frame, copies the old content, and installs the writable PTE in the faulting process's page table.

TLB Shootdown

smp_tlb_shootdown(va) stores the target VA in TLB_FLUSH_VA, then sends a broadcast IPI (vector 0x42) using the LAPIC All-Excluding-Self shorthand. Each AP's handle_tlb_flush handler reads the VA and executes invlpg, then signals EOI. This is a simple single-VA shootdown; a production kernel would batch flushes using a per-CPU bitmask and a generation counter.
