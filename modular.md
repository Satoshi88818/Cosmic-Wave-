CosmicWave v8 — Modular Source Tree

The v8 kernel source has been split from a single main.rs into focused modules. Each module owns one subsystem; main.rs is now only the entry point and test runner.

Project Layout

cosmicwave/ ├── Cargo.toml ├── x86_64-cosmicwave.json ├── build.sh └── src/ ├── linker.ld ├── main.rs ← feature gates, crate-level uses, _start, tests, panic/OOM ├── constants.rs ← every compile-time constant and the two address helpers ├── serial.rs ← UART 16550 driver and print!/println! macros ├── sync.rs ← Spinlock<T> / SpinlockGuard<T> and preempt_disable/enable ├── arch.rs ← irq_save/restore, MSR/CR helpers, PRNG, port I/O ├── mm.rs ← frame allocator, slab+large allocator, page tables, │ TLB shootdown, VmaRegion, AddressSpace, exception table, │ safe user accessors (copy_from/to_user, get/put_user_u64) ├── percpu.rs ← TssEntry, PercpuData (with full IST stacks), PER_CPU, │ RunQueue, APIC registration, IDT init, LAPIC, SMP bringup ├── task.rs ← TaskState, CpuContext, TaskInner, Task, ALL_TASKS, CURRENT, │ enqueue_task, resched, WaitQueue, alloc_kstack, spawn_kthread ├── sched.rs ← sched_pick_next, sched_steal, sched_idle_wait, │ switch_context, perform_context_switch, perform_first_run, │ schedule(), notify_zombie ├── trap.rs ← InterruptFrame, all isr_ist!/isr_sw! stubs, every exception │ handler, page-fault handler, kill_current_task ├── signal.rs ← sig module, SigAction, SignalInfo, sigreturn_trampoline, │ deliver_signal, handle_pending_signals ├── vfs.rs ← Vnode trait, RegularFile, PipeInner, PipeReadEnd/WriteEnd, │ DevNull, DevZero, OpenFile, FileTable, RamFs, ramfs_init ├── syscall.rs ← SyscallArgs, all validators (va_user, fd_valid, …), │ every sys_* function, handle_syscall_frame └── elf.rs ← ElfError, Elf64Ehdr/Phdr, load_elf, create_task_from_elf
